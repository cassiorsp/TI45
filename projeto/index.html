O que é Git?

Git é um sistema de controle de versão distribuído, usado principalmente no desenvolvimento de software para registrar as alterações no código-fonte ao longo do tempo.

O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte com controle de versão usando Git. Ele permite que desenvolvedores colaborem em projetos de software de forma eficiente.

Apresente 3 alternativas para o GitHub.

Três alternativas para o GitHub são: GitLab, Bitbucket e SourceForge.

Para que serve o commit do GitHub?

O commit no GitHub serve para salvar uma versão específica das alterações feitas no código, permitindo que essas alterações sejam registradas no histórico do projeto.

Para que serve o push origin?

O comando push origin serve para enviar as alterações locais do repositório para o repositório remoto, atualizando o projeto compartilhado na plataforma.

O que é pull request no GitHub?

O pull request no GitHub é uma solicitação para revisar e possivelmente mesclar as alterações propostas de um branch para outro, permitindo a colaboração e revisão de código.

Explique o que é versionamento.

Versionamento é o processo de gerenciar e controlar as diferentes versões de um software, permitindo que desenvolvedores acompanhem e revertam alterações conforme necessário.

Explique o que é fluxo de trabalho colaborativo no GitHub.

O fluxo de trabalho colaborativo no GitHub envolve a utilização de branches, commits, pull requests e revisões de código para permitir que vários desenvolvedores trabalhem juntos de maneira organizada e eficiente.
